# 分布式系统数据一致性、可用性与网络分区容错性

CAP 原则 https://mwhittaker.github.io/blog/an_illustrated_proof_of_the_cap_theorem/

CAP 定理又被成为布鲁尔定理，是加州大学计算机科学家埃里克・布鲁尔提出来的猜想，后来被证明成为 分布式计算领域公认的定理 CAP 定义，在高并发的场景下要做取舍，在大型集群中分区容错很难保证，一旦要确保容错性，那么就 会损失数据一致性和高可用特性。在 CAP 原则中，P 是不可避免的，所以可选的就是 AP 模型或者 CP 模型，取决于具体的业务对数据的要求。

# Consistency

分布式系统数据的一致性。强一致性必定会降低可用性（会阻塞），导致不能达到好的并发性。

# Availability

单节点的可用性。弱一致性可以提高可用性，但是降低了数据的准确性。

# Partition Tolerance

网络分区的容错性。不可避免的网络出错问题，必须容忍这样的错误。

# 高性能、高可用、高扩展系统架构

# CDN

通过动态 DNS 解析，将请求按区域划分并分发到就近的服务器（只存放静态资源）

# HttpDNS

DNS 防劫持

# 流量网关

nginx 集群流量网关服务，包括 WAF、定向流量分发、负载策略等，用到的框架有 Openresty（Nginx+Lua）、Kong 等

# 业务网关

鉴权、单点登录、限流等，用到的框架有 SpringSecurity、Oauth2、JWT、CAS、SpringCloud Gateway、Zuul、Spring Session 等。业务网关可能有多个。可以通过 Sentinel 进行资源隔离（通过限制不同 API 的 QPS 实现）。
