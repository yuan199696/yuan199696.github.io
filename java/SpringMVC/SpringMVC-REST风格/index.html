REST 是一种架构风格，应用包括但不限于 WEB 开发，在 WEB 应用中表现为与 HTTP 协议息息相关。可以总的来说：

RESTful API (具有 REST 风格的 API): 用 URL 定位资源，用 HTTP 动词（GET,HEAD,POST,PUT,PATCH,DELETE）描述操作，用响应状态码表示操作结果。-- 参考自博客 -[沧海一滴]

简言而知，客户端到服务端的请求需要一套标准的 URl 格式（如 http://localhost:8080/test?username=zhangsan）、标准的请求方式（Http 的 post 请求等）、标准的返回结果（特指状态码，如 404）

# 乱码问题

乱码问题解决：

需要设置 request 和 response 的编码方式，可以手动编写过滤器，也可以由现成的框架来实现。

对于 post 和 get 的编码方式不同：

post：必须要分别设置 request 和 response 的编码格式

get：在 tomcat 的 server.xml 文件中，添加 URIEncoding=utf-8

!> 在一个应用程序中可能包含多个过滤器，过滤器之前一般是没有顺序要求的，但是如果设置了编码过滤器，

那么要求编码过滤器必须放在所有过滤器的最上面，保证编码过滤器优先执行，否则编码过滤器将不起作用。

# 获取请求参数

在以往的 Sevelt 中，可以通过 request.getParameters (); 来获取请求参数的值，那么在 SpringMVC 中该如何获取呢？

# @RequestParam

默认情况下，可以直接在方法的参数中填写跟请求参数一样的名称，此时会默认接受请求的参数，如果有值，直接赋值给方法参数，如果没有，那么直接给空值。

当发送请求的时候，找到对应的处理方法之后，会根据方法参数的名称从 request 中获取对应的参数值，并封装到参数中，要求方法的参数必须和 url 中参数名字一致。如果两者名字不同，可以通过 @RequestParam（用来获取路径中？后面的参数值）中的值来和 url 中参数一致进行匹配。注意与 @PathVariable（用来获取路径中的值）的用法区别。

该注解参数值：

  •  value(或者name)：获取的参数值

  •  required：表示当前参数是否必须存在，默认为true，为true时，若请求中没有该参数则报错：400 bad request

  •  defaultValue：没有传递参数时使用默认值

# @RequestHeader

获取请求头参数，该注解参数值与 @RequestParam 相同。在 servlet 中为：request.getHeaders ();

# @CookieValue

获取 cookie。在 Servlet 中为：Cookie [] cookies = request.getCookies ();
