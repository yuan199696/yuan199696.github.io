# 将 Springboot 项目编译为镜像文件

参考

修改 Springboot 项目的 pom.xml

在项目的 src/mian 下新建包 docker，在 docker 下新建一个 Dockerfile 文件，内容为

FROM java:8
VOLUME /tmp
ADD ica-4.1.0.jar ica.jar
ENTRYPOINT ["java","-Djava.security.egd=file:/dev/./urandom","-jar","/ica.jar"]

FROM ：jdk 版本，如果本地没有该镜像，那么将自动下载 jdk

VOLUME ：指定了临时文件目录为 /tmp。其效果是在主机 /var/lib/docker 目录下创建了一个临时文件，并链接到容器的 /tmp。该步骤是可选的，如果涉及到文件系统的应用就很有必要了。/tmp 目录用来持久化到 Docker 数据文件夹，因为 Spring Boot 使用的内嵌 Tomcat 容器默认使用 /tmp 作为工作目录

ADD ：springboot 项目打包好的 jar（即项目 target 目录下打包好的 jar 的名字） 重命名.jar，代表将该 jar 添加到 docker 中作为镜像

ENTRYPOINT ：执行 jar。为了缩短 Tomcat 启动时间，添加一个系统属性指向 “/dev/./urandom” 作为 Entropy Source

在项目目录下先后执行 mvn clean、mvn install，接着执行 mvn package docker:build

报错：

[ERROR] Failed to execute goal com.spotify:docker-maven-plugin:1.0.0:build (default-cli) on project ica: Exception caught: java.util.concurrent.ExecutionException: com.spotify.docker.client.shaded.javax.ws.rs.ProcessingException: org.apache.http.conn.HttpHostConnectException: Connect to localhost:2375 [localhost/127.0.0.1, localhost/0:0:0:0:0:0:0:1] failed: Connection refused: connect -> [Help 1]

解决：勾选以下选项

再次执行 mvn package docker:build，报错：

Failed to execute goal com.spotify:docker-maven-plugin:1.0.0:build (default-cli) on project dockerDemo: Exception caught: java.util.concurrent.ExecutionException: com.spotify.docker.client.shaded.javax.ws.rs.ProcessingException: org.apache.http.client.ClientProtocolException: Cannot retry request with a non-repeatable request entity: Connection reset by peer -> [Help 1]

解决：artifactId 要小写

再次执行 mvn package docker:build，报错：

[ERROR] Failed to execute goal com.spotify:docker-maven-plugin:1.0.0:build (default-cli) on project ica: Exception caught: Get https://registry-1.docker.io/v2/: net/http: request canceled (Client.Timeout exceeded while awaiting headers) -> [Help 1]

解决：配置加速镜像

再次执行 mvn package docker:build，构建成功

# 后台启动
